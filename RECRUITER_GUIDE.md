# Portfolio Review Guide

## Two-minute navigation route

1. Start with the architecture and active repository table in `README.md`.
2. Open `dbt-analytics-engineering-lab` to inspect modeling and testing artifacts.
3. Open `semantic-layer-ai-agent-lab` to inspect governed AI analytics.
4. Open `supply-chain-decision-twin-agent` to inspect the Dify integration and decision-support boundary.
5. Open `mlops-llmops-evaluation-lab` to inspect shared release governance.
6. Select one domain repository: supply-chain operations or revenue growth.

## Evidence routes

| Evidence focus | Repository path |
|---|---|
| Analytics engineering | Cloud Warehouse → dbt → Semantic Agent |
| Data pipelines | Cloud Warehouse → Orchestration → Cloud Executable |
| BI engineering | BI Dashboard Lab → dbt → Supply Chain or Revenue |
| Supply-chain analytics | Control Tower → Decision Twin → Causal Experimentation |
| Revenue and growth | Revenue Growth → BI Dashboard → Causal Experimentation |
| AI-native analytics | Semantic Agent → Shared Evaluation → Decision Twin |
| Portfolio governance | Roadmap → one foundational repo → one AI-native repo → one domain repo |

## Reading the claims

“Complete” means complete for the repository's documented synthetic/local scope. It is not a claim of enterprise deployment, production scale, autonomous authority or measured real-world impact.
